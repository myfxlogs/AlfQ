# 20 - 错误码与异常处理规范

> 跨语言（Go / Python / TS）统一错误模型。**所有错误码集中在 `proto/alfq/v1/errors.proto` 单一源**；Go/Py/TS 各自生成 enum + helper。运行时错误 → Connect Code + ErrorDetail，记录到日志/审计/指标。

## 1. 错误模型

### 1.1 三层结构

```
Connect.Code (传输层)           # InvalidArgument, NotFound, ...
   │
   ▼
ErrorDetail (业务层)            # code, message, meta
   │
   ▼  内含
ErrCode enum (业务码)           # ALFQ_RISK_REJECTED, ALFQ_BROKER_DISCONNECTED, ...
```

## 2. 重试策略矩阵

| ErrCode | 客户端可重试 | 服务端自动重试 | 退避 |
|---|---|---|---|
| ERR_TIMEOUT / DEADLINE_EXCEEDED | ✓ | ✓ | 指数 250ms→4s |
| ERR_DEPENDENCY_UNAVAILABLE | ✓ | ✓ | 指数 |
| ERR_RATE_LIMITED | ✓（看 Retry-After） | ✗ | header 指定 |
| ERR_BROKER_TIMEOUT | ✓ | ✓ | 1s→16s 最多 3 次 |
| ERR_BROKER_DISCONNECTED | ✓ | ✓ | 重连后 |
| ERR_DB_DEADLOCK | ✗ | ✓ | 100ms 抖动 |
| ERR_DB_CONNECTION | ✓ | ✓ | 指数 |
| ERR_NATS_PUBLISH_FAILED | ✓ | ✓ + Outbox | — |
| ERR_VALIDATION_FAILED | ✗ | ✗ | — |
| ERR_PERMISSION_DENIED | ✗ | ✗ | — |
| ERR_IDEMPOTENCY_CONFLICT | 视情况 | ✗ | 重发同 key |
| ERR_RISK_REJECTED | ✗ | ✗ | — |
| ERR_ORDER_DUPLICATE | ✗ | ✗ | — |
| ERR_ORDER_INSUFFICIENT_MARGIN | ✗（业务） | ✗ | — |
| ERR_AI_PROVIDER_ERROR | ✓ | ✓ | 1s→8s |
| ERR_QUOTA_EXCEEDED | ✗ | ✗ | — |

服务端重试用 `cenkalti/backoff/v4`；客户端遵循 `Retry-After` header。

## 3. 各语言实现

### 3.1 Go

`backend/go/internal/common/errs/`：

```go
type Error struct {
    Code    pb.ErrCode
    Msg     string
    Meta    map[string]string
    Cause   error
}

func New(code pb.ErrCode, msg string, meta ...M) *Error { ... }
func Wrap(err error, code pb.ErrCode, msg string) *Error { ... }

// ToConnect 把 *Error 转 connect.Error（带 detail）
func ToConnect(err error) error { ... }

// Predicates
func IsRetryable(err error) bool { ... }
func Code(err error) pb.ErrCode { ... }
```

约定：
- 业务函数返回 `*errs.Error` 或 wrap 的错误
- handler 出口统一 `errs.ToConnect(err)`
- 日志中错误必带 `code` / `cause` 字段

### 3.2 Python

`research/alfq_research/errors.py`：

```python
class ALFQError(Exception):
    def __init__(self, code: ErrCode, msg: str, meta: dict | None = None):
        self.code = code
        self.msg = msg
        self.meta = meta or {}
        super().__init__(f"[{code.name}] {msg}")
```

Connect 客户端把响应错误解码为对应子类（`RiskRejected`、`Unauthenticated` 等）。

### 3.3 TypeScript

`frontend/src/lib/errors.ts`：

```ts
import { ConnectError, Code } from "@connectrpc/connect-web";
import { ErrCode, ErrorDetail } from "@/gen/alfq/v1/errors_pb";

export function parseError(e: unknown): { code: ErrCode; msg: string; meta: Record<string,string> } {
  if (e instanceof ConnectError) {
    const detail = e.findDetails(ErrorDetail)[0];
    if (detail) return { code: detail.code, msg: detail.message, meta: Object.fromEntries(detail.meta) };
  }
  return { code: ErrCode.ERR_INTERNAL, msg: String(e), meta: {} };
}
```

前端按 `code` 显示对应 i18n 消息。


绝不暴露内部错误细节（堆栈、SQL、文件路径）。

## 5. 指标与告警

- 所有错误自动计入 `alfq_rpc_requests_total{code}`
- 错误码 → severity 映射（用于告警）：
  - `ERR_VAULT_SEALED` / `ERR_RISK_KILLSWITCH_ACTIVE` / `ERR_BROKER_DISCONNECTED` → **page**
  - `2000-2099` 风控 → **info**（业务正常）
  - `5xx Internal` 比例突变 → **warning**

## 6. 审计

写操作失败时（PERMISSION_DENIED / RISK_REJECTED / VALIDATION_FAILED）必须写审计日志，含错误码与脱敏 payload。
