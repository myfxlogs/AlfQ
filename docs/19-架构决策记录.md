# 19 - 架构决策记录（ADR）

> 所有重大架构/技术/流程决策必须落 ADR。本文是 **模板 + 治理规则 + 首批 ADR 索引**。具体 ADR 实例位于 `docs/adr/` 目录。

## 1. 什么是 ADR

Architecture Decision Record，简短文档记录**一次重要决策**：
- 背景（为什么需要决策）
- 选项
- 决定
- 后果

借鉴：[Michael Nygard 的 ADR 模板](https://github.com/joelparkerhenderson/architecture-decision-record)。

## 2. 何时写 ADR（强制）

任何符合以下之一：

1. 引入新依赖（库 / 服务 / 第三方）
2. 修改对外协议或破坏向后兼容
3. 跨服务边界 / 模块接口的设计抉择
4. 重大性能 / 安全 / 合规相关决策
5. 数据模型重大变更
6. 部署形态 / 环境策略变更
7. 偏离已有 docs/0X 章节的实现
8. 弃用某项功能或路径

## 3. 命名与位置

- 路径：`docs/adr/NNNN-<slug>.md`
- 编号 4 位、单调递增（占用即不可回收）
- slug：kebab-case，简短描述
- 例：`docs/adr/0001-use-connect-rpc.md`

## 4. ADR 模板

```markdown
# NNNN - <标题：动词开头>

| 字段 | 值 |
|---|---|
| 日期 | YYYY-MM-DD |
| 状态 | proposed / accepted / superseded / deprecated |
| 决策者 | @user1, @user2 |
| 影响范围 | 服务/模块/全局 |
| 关联 ADR | #NNNN |
| 关联 docs | docs/0X-xxx.md |

## 背景（Context）
（2-5 段：为什么需要这个决策，问题是什么，约束有哪些）

## 选项（Options）
### A. <方案 A>
- 优点
- 缺点
- 风险

### B. <方案 B>
...

### C. <方案 C>
...

## 决策（Decision）
我们选择 **<方案 X>**。

理由：
- ...

## 后果（Consequences）
### 积极
- ...

### 消极 / 成本
- ...

### 跟进事项
- [ ] 任务 1
- [ ] 任务 2

## 验收
- [ ] 评审通过（≥ 2 名 reviewer + 必要 role）
- [ ] 关联代码变更引用本 ADR
- [ ] 相关 docs/0X 已更新
```

## 5. ADR 生命周期

```
proposed ──评审通过──► accepted ──新方案替代──► superseded by NNNN
                          │
                          └──不再需要──► deprecated
```

- **proposed**：起草中，PR 评审
- **accepted**：合入 main，开始执行
- **superseded**：被另一份 ADR 取代，文件标注 `Superseded by #NNNN`
- **deprecated**：不再使用，但不替代

**已 accepted 的 ADR 不可修改**（事实保留），需变更则新建 ADR 取代。

## 6. 评审要求

| 类型 | 必要 reviewer |
|---|---|
| 通用技术 | 2 名工程 + 架构 owner |
| 安全 / 合规 | + security_officer |
| 风控 / 资金路径 | + risk_officer |
| 法务相关 | + legal |
| 涉及租户数据 | + privacy_officer |

CODEOWNERS 文件覆盖 `docs/adr/` 路径要求 2 名 + 上述角色。

## 7. 与代码的关联

- 代码引用 ADR 用注释：

```go
// ADR-0007: 使用 sqlc 而非 ORM
//go:generate sqlc generate
```

- ADR 关联 issue / PR：在 ADR 末尾列出
- 实施完成后在 ADR 添加 `## 实施记录` 段（accepted 后允许追加此段）

## 9. AI Agent 写 ADR 的要求

- Agent 实施前发现 docs 未覆盖的决策 → 必须先开 ADR PR
- ADR 评审通过后再开实施 PR
- 实施 PR 必须在 commit message 与 README 引用 ADR
- 禁止 Agent 自行做"小聪明"绕过 ADR
