## 0. 一句话总结

> **历史行情是地基，宏观是装饰，AI 是脚手架。**
> 地基不稳，装饰和脚手架都是负债。

ALFQ 当前方向正确：先把**数据 + 执行**做到不出错；AI 用最朴素形态（ONNX 推理 + DSL 因子）跑通；待真金白银验证 alpha 后，再决定要不要叠加宏观和更复杂的 AI。

## 1. 稳定量化系统的真正基础（按重要性排序）

### 1.1 数据可信（Ground Truth） ★ 最重要

- **同源同口径**：回测用的 bar 与实盘用的 tick 必须**同经纪商、同 symbol、同时区、同价格类型**（bid/ask/mid 不混）
- **可追溯**：每条 tick/bar 留 `(broker, symbol_raw, canonical, ts_unix_ms, arrived_unix_ms)`，事后可还原
- **质量可观测**：gap / outlier / clock skew 指标 + 告警，数据坏了**当天**就知道
- **历史与实时一致**：backfill 历史 bar 与同段时间实时聚合 bar 误差 < 1 tick

> ✅ ALFQ 现状：DP-1~7 + SM-1~3 已搭好，**最值钱的资产**。

### 1.2 执行确定性

- **下单路径强同步**：API → 风控 → OMS 同进程内（ADR-0010 核心），跨进程 = 延迟分布不可预测
- **状态机单向**：order/strategy/position 仅 `pending→open→closed`，禁止回退
- **幂等**：同信号重发不出双单（ticket / client_order_id 唯一约束）
- **可回滚**：Kill Switch < 1s 砍掉所有在途单
- **会计可对账**：本地 PnL 与经纪商对账单日级 100% 一致

### 1.3 回测—实盘一致性（Consistency Gate）

- 向量化 vs 事件驱动 corr ≥ 0.95、日 PnL MAD < 1%
- 没有这道闸 = 研究端 bug 直接喂给真钱
- ALFQ 现状 corr=0.972 / MAD=0.0042% ✓

### 1.4 故障隔离与可恢复

- 行情中断不影响下单状态查询（DB 是 source of truth）
- 单策略熔断不影响其他策略
- 进程崩溃自动重启（QW-1 已做）
- 断连后**对账重连**，而非裸跑（OS-1/OS-2 待做）

### 1.5 风控前置

- 下单前必过 N 条同步规则（仓位、回撤、品种白名单、min_lot/lot_step）
- 失败 = reject，非 warn

### 1.6 可观测性

- 三层 SLO：行情（tick latency p99）/ 订单（submit p99 + 成功率）/ 数据（CH 写入成功率）
- 看板第一眼能判断系统健康，不需翻日志

### 1.7 容易高估的事（**不重要**）

- 策略本身的"聪明程度"——稳定系统跑 60 分策略 > 不稳定系统跑 90 分策略
- 微秒级延迟（除非做 HFT，否则 p99 < 150ms 已够）
- AI/ML 模型本身——只是策略一种实现，不是基础
- 漂亮前端——能看清 PnL 和持仓即可
- 微服务拆得多细（参考 ADR-0010）

## 2. 数据源的三层金字塔

```
        ┌─────────────────────┐
        │  替代数据 (alt-data) │  5–10%   卫星 / 舆情 / 订单簿微结构
        │                     │          (HFT/对冲基金；中小团队 ROI 极低)
        ├─────────────────────┤
        │  宏观 / 基本面       │  20–30%  利率 / CPI / 财报 / 持仓报告
        │                     │          (中低频策略加分项，不是地基)
        ├─────────────────────┤
        │  历史行情 + 衍生量    │  60–70%  OHLCV / tick / orderbook / 波动率
        │                     │          ★ 地基，先做扎实
        └─────────────────────┘
```

### 2.1 为何先压历史行情

| 维度 | 历史行情 | 宏观/基本面 |
|---|---|---|
| 数据量 | 海量、高频、规整 | 稀疏、低频、口径乱 |
| 可回测 | 时间戳精确到 ms，无 look-ahead | 发布时间 vs 标的时间易污染 |
| 信号→执行延迟 | μs–ms | 分钟–小时 |
| 验证成本 | 一致性 gate 可跑 | 样本太少，统计显著性差 |
| ALFQ 现状 | ✅ 已建 | ❌ 待接 |

### 2.2 宏观什么时候值得做（按性价比排序）

1. **交易日历 / 节假日** ★★★★★（**这不是宏观，是基础设施**）—— 避免低流动性时段下单
2. **重大经济数据发布窗口** ★★★★（NFP / CPI / FOMC ±15min 暂停下单 / 收紧风控）—— 一张时间表就值得做
3. **利差 / 隔夜利率** ★★★（已隐含在 `broker_symbols.swap_long/short`）
4. **VIX / DXY 等宏观指数作为特征** ★★（只在 Sharpe > 1 后再加，否则易过拟合）
5. **新闻舆情 / 财报** ★（外汇相关性弱；做股票才有意义）

> **对外汇为主的 ALFQ**：1、2 必做（中期）；3 已有；4 选做；5 可忽略。

## 3. AI 在量化里的真实作用

### 3.1 AI **能**做什么（实测有效）

| 用途 | 价值 | 难度 | ALFQ 对应 |
|---|---|---|---|
| 非线性特征组合（树模型 / GBM）| ★★★ 因子已有 alpha 前提下，能榨出 10–30% 增量 | 中 | EP-1 已做 |
| 动态止盈止损 / 仓位调节（RL）| ★★★ 比固定阈值好 | 高 | 暂不做 |
| 市场状态分类（HMM / 聚类做 regime detection）| ★★★ 给策略切换提供依据 | 中 | 中期 |
| 异常 tick 检测 | ★★ 5σ 规则的加强版 | 低 | DP-5 已有基础 |
| 回测加速（向量化 → GPU）| ★★ 工程加分，不增 alpha | 低 | RP-2 已向量化 |
| LLM 做日志 / 告警摘要 | ★★ 运维提效，不参与决策 | 低 | assistant-svc 留 |
| LLM 做策略代码生成 | ★ 助手而非作者 | 中 | 慎用 |

### 3.2 AI **不能**做什么（被高估）

| 误区 | 真相 |
|---|---|
| "丢 K 线给 Transformer，自动学出策略" | 金融 SNR < 1%，学到的多半是过拟合或微观噪声 |
| "GPT-4 分析新闻 → 直接下单" | 延迟 + 一致性 + 可回测性全不达标 |
| "深度学习一定比线性模型强" | 树/线性在多数频率上不输 LSTM/Transformer，且更鲁棒、可解释 |
| "用 AI 就不需要人为定义因子" | 端到端从原始价格学，绝大多数失败；带先验的特征工程仍是基础 |

### 3.3 AI 在 ALFQ 数据流中的合理位置

```
原始 tick / bar
   │
   ▼
[人工 + DSL] 因子计算（RSI, MA, ATR, …）        ← 主体逻辑，可解释
   │
   ▼
[AI: 树模型 / 线性模型] 因子 → 信号 (-1 / 0 / 1) ← EP-1 已做这步
   │
   ▼
[确定性规则] 仓位 / 止损 / 风控                  ← 别让 AI 碰这里
   │
   ▼
[人工双签] paper → live                          ← 别让 AI 碰这里
```

> **核心原则**：AI 越靠近"决策出口"，事故损失越大。
> 把 AI 限制在 **特征 → 信号**；**信号 → 下单**完全由确定性规则负责。
> 这正是 ADR-0013 + EP-3 当前的做法。

### 3.4 经验比例（粗）

成熟量化团队 alpha 来源分布：

```
60–70%  数据 + 因子设计（人脑 + 历史行情）
15–25%  执行优化（滑点 / 成本控制）
 5–15%  AI / ML（非线性拟合 + 状态切换）
 5%     其他（宏观过滤 / 舆情）
```

**反直觉但常见**：

- **简单均值回归 + 严格风控** 长期可能跑赢 **复杂 DL + 松散风控**
- 数据干净度提升 10% > 换模型带来的收益
- 加一道交易日历过滤，可能就避开 30% 的爆仓案例

## 4. 对 ALFQ 的具体建议

### 4.1 现在（A–E 已完成）

1. **AI 比例先压在 < 20%**：ONNX 通道保留（ADR-0013 fallback），首批 paper 策略用 DSL 因子 + 简单线性/树模型即可
2. **不追深度学习**：当前数据量（10 年 1h bar ≈ 87k 条）远不够支撑 Transformer，过拟合是必然
3. **暂不接宏观数据源**：完全合理
4. **聚焦完成**：方案 D（MT Session Hub）+ 订单本地化收口，让"执行确定性"和"故障隔离"达到生产级

### 4.2 中期（首批策略 paper 验证后）

1. **加 `economic_calendar` 表**：数据源 ForexFactory / TradingEconomics API，高影响事件前后封盘
   - 一周工作量，alpha 收益 > 任何 AI 调参
2. **AI 真正花力气的方向**（辅助系统，不直接产信号）：
   - 异常检测：DP-5 基础上加 LightGBM 分类
   - Regime 切换：HMM 把市场分成 3–4 个状态，给不同状态绑不同策略

### 4.3 长期（一年后 + Sharpe 已稳定）

- 宏观特征作为 **regime classifier 的输入**，而非策略入场信号
- 策略组合扩到 > 20 个时，AI 做组合优化（仓位分配）
- 替代数据 / 舆情 / LLM 推理 —— 前面基础长期稳定之后再碰

## 5. 决策守则（PR 评审时回头看这条）

| 出现下列情况 | 应当 |
|---|---|
| 新方案宣称用更复杂的 AI 提升收益 | 先问：数据干净度还能提升吗？执行确定性还有漏洞吗？这两个不解决，AI 提升基本是噪声 |
| 想接入新数据源（卫星、舆情、订单簿）| 先问：现有历史行情的 alpha 榨干了吗？回测一致性还守得住吗？ |
| 想拆服务 / 加微服务 | 看 ADR-0010，违反则拒 |
| 想让 AI 直接决定下单参数 | 拒。AI 只到信号；下单参数走确定性规则 |
| 想跳过双签上 live | 拒。任何理由 |
| 想绕过一致性 gate | 拒。任何理由 |
