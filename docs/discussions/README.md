# discussions/ — 讨论记录与回顾

> 用途：保存与 Cascade / DeepSeek 等 AI Agent 的**方向性讨论**沉淀，便于事后回顾。
>
> 性质区分：
>
> | 目录 | 性质 | 是否约束执行 |
> |---|---|---|
> | `docs/adr/` | 架构决策记录 | ✅ 是（违反需先改 ADR）|
> | `docs/tasks/` | 任务清单 / 执行计划 | ✅ 是（AI Agent 按其推进）|
> | `docs/enhancements/` | 具体优化设计 / 增强提案 | ✅ 是（按其落地）|
> | **`docs/discussions/`** | **判断依据 / 方向性备忘 / 思路沉淀** | **❌ 否（仅供参考）** |
>
> 本目录的文档：
> - **不是任务清单**——不要按它做事，做事看 `tasks/` 和 `enhancements/`
> - **不是 ADR**——没有"已生效不可逆"的约束力
> - **是回顾用的**——做新功能 / 砍方向 / 评估 PR 前回来翻一翻

## 写新讨论文档的轻量约定

1. 顶部加 frontmatter 风格的元信息（日期 / 作者 / 性质 / 关联 ADR & 任务）
2. **§0 一句话总结**——回顾时只看这一段就能恢复上下文
3. 结尾给"决策守则"或"行动建议"——便于以后照单办事
4. 篇幅不强求短，但**结论必须能两句话说清**
