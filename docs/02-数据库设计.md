# 02 - 数据库设计

> 涵盖 **PostgreSQL（交易主数据）**、**ClickHouse（时序）**、**Redis（缓存/会话）** 三套库的完整 DDL、索引、分区、迁移策略。AI Agent 实施时按本文件创建数据库结构。

## 0. 通用约定

- 所有 PG 表带 `tenant_id UUID NOT NULL`，启用 Row-Level Security
- 所有 PG 表带 `created_at / updated_at TIMESTAMPTZ DEFAULT now()`
- 所有"业务唯一键"采用 **UUID v7**（按时间有序）
- 所有 CH 表 `ORDER BY (tenant_id, symbol, ts)` 起手，按月分区
- 时间统一 **UTC**，存储为 PG `TIMESTAMPTZ` / CH `DateTime64(9, 'UTC')`（纳秒）
- 金额用 `NUMERIC(20,8)`；手数 `NUMERIC(12,4)`；价格 `NUMERIC(20,8)`

### 1.1 租户与用户

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE tenants (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name            TEXT NOT NULL UNIQUE,
  display_name    TEXT NOT NULL,
  status          TEXT NOT NULL DEFAULT 'active', -- active/suspended/deleted
  plan            TEXT NOT NULL DEFAULT 'basic',
  max_users       INT  NOT NULL DEFAULT 5,
  max_strategies  INT  NOT NULL DEFAULT 20,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  email           CITEXT NOT NULL,
  password_hash   TEXT NOT NULL,            -- argon2id
  totp_secret     TEXT,                     -- encrypted by Vault Transit
  status          TEXT NOT NULL DEFAULT 'active',
  last_login_at   TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(tenant_id, email)
);

CREATE TABLE roles (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id   UUID REFERENCES tenants(id) ON DELETE CASCADE, -- NULL = system role
  name        TEXT NOT NULL,
  description TEXT,
  is_system   BOOLEAN NOT NULL DEFAULT FALSE,
  UNIQUE(tenant_id, name)
);

CREATE TABLE permissions (
  id     UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code   TEXT NOT NULL UNIQUE,          -- e.g. strategy.deploy, account.read
  name   TEXT NOT NULL
);

CREATE TABLE role_permissions (
  role_id       UUID REFERENCES roles(id) ON DELETE CASCADE,
  permission_id UUID REFERENCES permissions(id) ON DELETE CASCADE,
  PRIMARY KEY (role_id, permission_id)
);

CREATE TABLE user_roles (
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  role_id UUID REFERENCES roles(id) ON DELETE CASCADE,
  PRIMARY KEY (user_id, role_id)
);

-- 资源级 ACL：覆盖 RBAC 默认
CREATE TABLE acls (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id    UUID NOT NULL,
  user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  resource_type TEXT NOT NULL,    -- strategy / account / factor
  resource_id  UUID NOT NULL,
  action       TEXT NOT NULL,     -- read / write / execute
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, resource_type, resource_id, action)
);

CREATE INDEX idx_users_tenant ON users(tenant_id);
CREATE INDEX idx_acls_resource ON acls(resource_type, resource_id);
```

### 1.9 行级安全（RLS）

对所有带 `tenant_id` 的业务表启用：

```sql
ALTER TABLE accounts ENABLE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON accounts
  USING (tenant_id = current_setting('app.tenant_id', true)::uuid);
-- 应用层每次连接 SET app.tenant_id = '<uuid>';
```

对**所有**含 `tenant_id` 的表都重复此策略（脚本生成）。`super_admin` 通过 `SET ROLE` 切换 bypass 角色，需审计。

### 2.7 多租户隔离（CH）

CH 不支持 PG 风格 RLS，采用：

1. 应用层强制 `WHERE tenant_id = ?`（gRPC interceptor 注入）
2. 为每租户创建独立 CH `user` + `ROW POLICY`：

```sql
CREATE ROW POLICY tenant_pol ON alfq.ticks FOR SELECT
  USING (tenant_id = currentUser() = 'svc') AS PERMISSIVE TO tenant_xxx_user
  USING (tenant_id = toUUID('<tid>'));
```

## 3. Redis 键命名规范

## 6. 备份与归档

> **本章列出与本文档同存的核心表完整 DDL。后续文档新增的表为减少重复，仅在 §6.5 给出索引；请到原文档查看 DDL。**

### 6.1 备份策略

| 数据 | 策略 |
|---|---|
| PG | `pg_basebackup` 每日全量 + WAL 持续；保留 30 天；每月归档到对象存储 |
| CH | `BACKUP TABLE` 每周全量到 S3 + 跨副本复制 |
| Redis | RDB 每日 + AOF；缓存可丢，会话/锁丢失影响小 |
| MinIO | 跨区域复制 |
| 审计日志 | PG → S3 冷归档，保留 5 年，不可删除（对象锁） |
