# 增量优化方案目录 (Enhancements)

本目录用于存放针对已有系统的**增量优化与改进方案**文档，与 `docs/` 根目录下的"基线架构文档"（01-29 系列）相区分。

## 与其他目录的关系

| 目录 | 定位 |
|---|---|
| `docs/01-29-*.md` | 系统基线架构与规范，相对稳定 |
| `docs/enhancements/` | **本目录** — 针对现有模块的增量优化方案，可包含设计、评审、验收标准 |
| `docs/refactor/` | 大规模重构提案与竣工报告 |
| `docs/adr/` | 架构决策记录（ADR），只记录"决策"，不含详细落地步骤 |
| `docs/tasks/` | 任务清单、Runbook、Roadmap |
| `docs/handover/` | 阶段性交接文档 |

## 文档命名规范

```
YYYY-MM-DD-<module>-<short-title>.md
```

例如：
- `2026-05-20-order-sync-incremental-design.md`
- `2026-06-01-risk-engine-rule-cache.md`

## 文档结构建议

每份增量优化文档建议包含：

1. **背景与问题陈述**：当前痛点、量化指标
2. **目标与非目标**：要解决什么 / 不解决什么
3. **方案设计**：数据流、API、表结构、关键算法
4. **替代方案对比**：为什么选这个方案
5. **实施步骤**：分阶段落地计划
6. **风险与回滚**：失败时如何回退
7. **验收标准**：可量化的成功指标
