## 4. 数据访问层

### 4.1 Connect Transport

```ts
// app/transport.ts
import { createConnectTransport } from "@connectrpc/connect-web";
import { Code, ConnectError, type Interceptor } from "@connectrpc/connect-web";
import { useAuthStore } from "@/stores/auth";

const authInterceptor: Interceptor = (next) => async (req) => {
  const token = useAuthStore.getState().accessToken;
  if (token) req.header.set("Authorization", `Bearer ${token}`);
  req.header.set("X-Trace-Id", crypto.randomUUID());
  try {
    return await next(req);
  } catch (e) {
    if (e instanceof ConnectError && e.code === Code.Unauthenticated) {
      await useAuthStore.getState().refresh();
      // 重试一次
      return await next(req);
    }
    throw e;
  }
};

export const transport = createConnectTransport({
  baseUrl: import.meta.env.VITE_API_BASE ?? "/api",
  interceptors: [authInterceptor],
  useBinaryFormat: false,        // 浏览器开发期用 JSON 便于调试
});
```

### 4.2 Query Client 与 RPC Hooks

```ts
// 示例：strategies 列表
import { useQuery } from "@tanstack/react-query";
import { createClient } from "@connectrpc/connect";  // 核心 API（类型/客户端）；transport 从 connect-web 导入
import { StrategyService } from "@/gen/alfq/v1/strategy_connect";
import { transport } from "@/app/transport";

const client = createClient(StrategyService, transport);

export function useStrategies() {
  return useQuery({
    queryKey: ["strategies"],
    queryFn: () => client.listStrategies({ page: { pageSize: 50 } }),
  });
}
```

可直接用 `@connectrpc/connect-query` 简化（自动生成 hook）。

### 4.3 SSE 流（订单事件 / 行情 / 风控告警）

```ts
// lib/stream.ts
export async function subscribe<T>(
  call: AsyncIterable<T>,
  onMsg: (m: T) => void,
  signal: AbortSignal
) {
  try {
    for await (const msg of call) {
      if (signal.aborted) break;
      onMsg(msg);
    }
  } catch (e) {
    if (!signal.aborted) throw e;
  }
}

// 用法
useEffect(() => {
  const ac = new AbortController();
  subscribe(
    client.streamOrders({ accountId }, { signal: ac.signal }),
    (evt) => qc.setQueryData(["orders"], updateFn(evt)),
    ac.signal
  );
  return () => ac.abort();
}, [accountId]);
```

## 5. 状态管理

| 状态 | 工具 |
|---|---|
| 服务器数据 | TanStack Query（缓存/重试/失效） |
| 全局 UI 状态（主题、侧栏） | Zustand |
| 认证（token、用户、租户） | Zustand + localStorage（access）+ httpOnly cookie（refresh） |
| 表单 | react-hook-form + zod |
| 通知 toast | sonner |

**refresh_token 存储**：建议放 **HttpOnly + Secure + SameSite=Strict cookie**（服务端写入），access_token 放内存（不放 localStorage 避免 XSS 窃取）。AI Agent 实施时按此安全模型来。

## 6. 关键页面规约

### 6.1 Dashboard
- 卡片：今日 PnL、未平仓数、活跃策略数、近 1h 错误数
- 图表：净值曲线（按账户/全部）、回撤曲线
- 列表：最新告警、最新成交

### 6.2 策略详情
- Tabs：参数 / 部署 / 实时信号 / 历史回测 / 日志
- 实时信号：用 `streamDeploymentEvents` 推送
- 操作按钮：启停、改参数（敏感操作需 TOTP）

### 6.3 因子编辑
- 表达式输入框（Monaco Editor + 语法高亮，DSL 模式）
- "校验"按钮 → `ValidateExpression` RPC
- "预览"按钮 → `PreviewFactor` 返回 200 根历史 bar 上的因子曲线（lightweight-charts）
- "提交审批"按钮（草稿→待审批）

### 6.4 Kill Switch
- 单独页 + 红色警告
- 二次输入 TOTP + 输入"CONFIRM"字符串
- 操作记录展示（最近 5 次）

### 6.5 回测
- 创建表单：策略 / 时间段 / 品种 / 参数
- 运行中：`StreamBacktestLogs` 实时日志
- 完成后：iframe 嵌入 quantstats HTML 报告 + 关键指标卡片

## 10. 安全要点

- CSP 严格策略：`default-src 'self'`，禁内联脚本
- 所有 XSS 攻击面用 React 默认转义；富文本必须 DOMPurify
- 表单 zod 校验
- 上传文件：限制类型/大小
- 路由守卫与服务端权限**双重**校验（前端只是 UX，权威在后端）
