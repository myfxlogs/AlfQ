## 3. 架构变更摘要

### 3.1 Bootstrap 统一启动

4 个 `cmd/*/main.go` 从各自散落的配置加载/日志/pgpool/redis/nats 初始化统一为：

```go
func main() {
    bootstrap.Run("svc-name", register, opts...)
}
```

引入 `internal/common/bootstrap` 包（Functional Options + ServeMuxAdapter 模式），CORS 中间件提升到 bootstrap。

### 3.2 服务级 Runner 模式

各服务业务逻辑从 `main.go` 的 `register` 函数提取到独立 runner：

```
internal/tradingcore/runner.go   → RunTradingCore
internal/quantengine/runner.go   → RunQuantEngine
internal/assistantsvc/runner.go  → RunAssistant
internal/mdgateway/              → RunGateway
```

### 3.3 adminapi 拆分

`service.go` 从 330 行单文件拆为 6 个 handler + adapter：

| 文件 | 职责 |
|------|------|
| `service.go` (43行) | Service 构造 |
| `auth_handler.go` | 认证 |
| `strategy_handler.go` | 策略 |
| `account_handler.go` | 账户 |
| `broker_handler.go` | 经纪商 |
| `backtest_handler.go` | 回测 |
| `audit_handler.go` | 审计 |
| `adapter.go` | proto↔domain 转换 |

### 3.4 依赖清理

- 移除 ClickHouse 直接依赖（`ch-go`、`clickhouse-go/v2`）
- 移除 Vault 依赖（`vault/api` 及 10+ 间接依赖）
- 移除 OpenTelemetry 依赖（`otel`、`otel/trace`）
- 删除未使用包：`flags`、`idempotency`、`interceptor`、`logger`、`telemetry`、`vault`
- go.mod 间接依赖净减少 ~20 个

### 3.5 共享构建

4 个 `Dockerfile` 合并为 1 个 `Dockerfile.builder`，通过 `--build-arg SVC=<name>` 参数化。
